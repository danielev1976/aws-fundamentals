## Vad är Amazon S3?

**Amazon Simple Storage Service (S3)** är en objektlagringstjänst från AWS som låter dig lagra och hämta godtyckliga mängder data när som helst, var som helst på webben.

Till skillnad från ett filsystem på en dator organiserar S3 data som **objekt** i **buckets** (hinkar). Varje objekt består av:

- **Data** – själva filinnehållet (bild, PDF, JSON, video, etc.)
- **Nyckel (Key)** – ett unikt namn inom bucketen (fungerar som en filsökväg, t.ex. `uploads/users/avatar-123.png`)
- **Metadata** – valfri information om objektet (content-type, skapandedatum, etc.)

### Varför S3?

| Egenskap | Beskrivning |
|---|---|
| Skalbarhet | Lagrar obegränsat med data |
| Tillgänglighet | 99.999999999% (11 nior) hållbarhet |
| Kostnad | Betala bara för det du använder |
| Integration | Sömlös integration med övriga AWS-tjänster |
| Säkerhet | Finkornad åtkomstkontroll via IAM och bucket policies |

## Grundläggande Begrepp

### Bucket

En bucket är en behållare för objekt. Varje bucket har ett **globalt unikt namn** över hela AWS. Du väljer också i vilken **region** (geografisk plats) bucketen ska skapas – välj nära dina användare för lägst latens.

```
s3://min-applikation-uploads/
```

### Object Key

Nyckeln är objektets "sökväg" i bucketen. S3 är egentligen platt (ingen riktig mappstruktur), men `/` i nyckelnamn ger en mappliknande upplevelse i AWS Console.

```
uploads/images/profil-bilder/user-42.jpg
uploads/documents/2024/rapport.pdf
```

### Åtkomstkontroll

S3 har flera lager av åtkomstkontroll:

- **Block Public Access** – standardinställning som blockerar all offentlig åtkomst
- **Bucket Policy** – JSON-baserade regler för vilka som får göra vad
- **IAM Roles/Policies** – styr vilka AWS-resurser (EC2, Lambda, etc.) som får använda bucketen
- **Pre-signed URLs** – tillfälliga, signerade URL:er för tidsbegränsad åtkomst

**Frontend (React/Vue)**
- Tillåter användaren att ladda upp filer

**HTTP API (API Gateway)**
- Tar emot HTTP-anrop från frontend
- Routar anrop till rätt Lambda-funktion

**Lambda-funktioner**
- Hanterar lättviktiga operationer: generera pre-signed URLs, validering

- Kommunicerar direkt med S3 via AWS SDK

**S3 Bucket**
- Lagrar alla filer/objekt
- Tillgänglig från både Lambda och EC2

## S3 och Lambda: Pre-signed URLs

Ett vanligt mönster är att låta **frontend ladda upp direkt till S3** utan att filen behöver passera genom din backend. Detta kallas **pre-signed URL upload**.

### Flöde

```
1. Frontend ───GET /upload-url?filename=bild.jpg──▶ Lambda
2. Lambda  ───genererar pre-signed PUT URL──────▶ S3
3. Lambda  ◀──returnerar URL (giltig 15 min)──────
4. Frontend ◀──{ uploadUrl: "https://s3.aws..." }──
5. Frontend ───PUT bild.jpg direkt till S3────────▶ S3
```

### Lambda-funktion: Generera Pre-signed URL (Node.js)

```javascript
// generate-upload-url/index.mjs
import { S3Client, PutObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";
import { randomUUID } from "crypto";

const s3 = new S3Client({ region: process.env.AWS_REGION });
const BUCKET_NAME = process.env.S3_BUCKET_NAME;

export const handler = async (event) => {
  const { filename, contentType } = JSON.parse(event.body);

  // Säker nyckel – använd aldrig användarens filnamn direkt!
  const key = `uploads/${randomUUID()}-${filename}`;

  const command = new PutObjectCommand({
    Bucket: BUCKET_NAME,
    Key: key,
    ContentType: contentType,
  });

  // URL giltig i 15 minuter
  const uploadUrl = await getSignedUrl(s3, command, { expiresIn: 900 });

  return {
    statusCode: 200,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ uploadUrl, key }),
  };
};
```

### Lambda-funktion: Hämta Pre-signed URL för nedladdning (Node.js)

```javascript
// generate-download-url/index.mjs
import { S3Client, GetObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const s3 = new S3Client({ region: process.env.AWS_REGION });

export const handler = async (event) => {
  const { key } = event.queryStringParameters;

  const command = new GetObjectCommand({
    Bucket: process.env.S3_BUCKET_NAME,
    Key: key,
  });

  const downloadUrl = await getSignedUrl(s3, command, { expiresIn: 3600 });

  return {
    statusCode: 200,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ downloadUrl }),
  };
};
```

## S3 och Spring Boot: AWS SDK för Java

Din Spring Boot-backend på EC2 interagerar med S3 via **AWS SDK for Java v2**.

### S3-konfiguration (Spring Boot)

```java
// config/S3Config.java
@Configuration
public class S3Config {

    @Value("${aws.region}")
    private String region;

    @Bean
    public S3Client s3Client() {
        return S3Client.builder()
                .region(Region.of(region))
                // På EC2 med IAM Role behövs ingen explicit autentisering!
                // SDK:n hämtar credentials automatiskt från EC2 Instance Metadata
                .credentialsProvider(DefaultCredentialsProvider.create())
                .build();
    }
}
```

### S3-tjänst (Spring Boot)

```java
// service/S3Service.java
@Service
public class S3Service {

    private final S3Client s3Client;

    @Value("${aws.s3.bucket-name}")
    private String bucketName;

    public S3Service(S3Client s3Client) {
        this.s3Client = s3Client;
    }

    /**
     * Ladda upp en fil till S3
     */
    public String uploadFile(String key, byte[] data, String contentType) {
        PutObjectRequest request = PutObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .contentType(contentType)
                .build();

        s3Client.putObject(request, RequestBody.fromBytes(data));
        return key;
    }

    /**
     * Ladda ned en fil från S3
     */
    public byte[] downloadFile(String key) {
        GetObjectRequest request = GetObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build();

        ResponseBytes<GetObjectResponse> objectBytes = s3Client.getObjectAsBytes(request);
        return objectBytes.asByteArray();
    }

    /**
     * Ta bort ett objekt från S3
     */
    public void deleteFile(String key) {
        DeleteObjectRequest request = DeleteObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build();

        s3Client.deleteObject(request);
    }

    /**
     * Lista alla objekt med ett visst prefix (t.ex. alla filer för en användare)
     */
    public List<String> listFiles(String prefix) {
        ListObjectsV2Request request = ListObjectsV2Request.builder()
                .bucket(bucketName)
                .prefix(prefix)
                .build();

        ListObjectsV2Response response = s3Client.listObjectsV2(request);
        return response.contents().stream()
                .map(S3Object::key)
                .collect(Collectors.toList());
    }
}
```

### REST-kontroller (Spring Boot)

```java
// controller/FileController.java
@RestController
@RequestMapping("/api/files")
public class FileController {

    private final S3Service s3Service;

    public FileController(S3Service s3Service) {
        this.s3Service = s3Service;
    }

    @PostMapping("/upload")
    public ResponseEntity<Map<String, String>> uploadFile(
            @RequestParam("file") MultipartFile file) throws IOException {

        String key = "processed/" + UUID.randomUUID() + "-" + file.getOriginalFilename();
        s3Service.uploadFile(key, file.getBytes(), file.getContentType());

        return ResponseEntity.ok(Map.of("key", key, "status", "uploaded"));
    }

    @GetMapping("/download/{key}")
    public ResponseEntity<byte[]> downloadFile(@PathVariable String key) {
        byte[] data = s3Service.downloadFile(key);

        return ResponseEntity.ok()
                .header(HttpHeaders.CONTENT_DISPOSITION, "attachment; filename=\"file\"")
                .body(data);
    }

    @DeleteMapping("/{key}")
    public ResponseEntity<Void> deleteFile(@PathVariable String key) {
        s3Service.deleteFile(key);
        return ResponseEntity.noContent().build();
    }
}
```

## Frontend: Uppladdning med Pre-signed URL

```javascript
// services/fileService.js (Vue/React – fungerar i båda)

const API_BASE = "https://din-api-gateway-url.amazonaws.com";

/**
 * Steg 1: Hämta en pre-signed URL från Lambda
 * Steg 2: Ladda upp filen direkt till S3
 */
export async function uploadFile(file) {
  // 1. Be Lambda om en uppladdnings-URL
  const urlResponse = await fetch(`${API_BASE}/upload-url`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      filename: file.name,
      contentType: file.type,
    }),
  });

  const { uploadUrl, key } = await urlResponse.json();

  // 2. Ladda upp direkt till S3 med den pre-signerade URL:en
  const uploadResponse = await fetch(uploadUrl, {
    method: "PUT",
    headers: { "Content-Type": file.type },
    body: file, // Skicka filen direkt – ingen base64-kodning behövs
  });

  if (!uploadResponse.ok) {
    throw new Error("Uppladdning misslyckades");
  }

  return key; // Spara nyckeln i din databas för framtida hämtning
}

/**
 * Hämta en nedladdnings-URL för att visa/ladda ned en fil
 */
export async function getDownloadUrl(key) {
  const response = await fetch(
    `${API_BASE}/download-url?key=${encodeURIComponent(key)}`
  );
  const { downloadUrl } = await response.json();
  return downloadUrl;
}
```

### React-komponent: Filuppladdning

```jsx
// components/FileUpload.jsx
import { useState } from "react";
import { uploadFile } from "../services/fileService";

export default function FileUpload() {
  const [status, setStatus] = useState("idle");
  const [uploadedKey, setUploadedKey] = useState(null);

  const handleFileChange = async (e) => {
    const file = e.target.files[0];
    if (!file) return;

    setStatus("uploading");
    try {
      const key = await uploadFile(file);
      setUploadedKey(key);
      setStatus("success");
    } catch (error) {
      console.error(error);
      setStatus("error");
    }
  };

  return (
    <div>
      <input type="file" onChange={handleFileChange} disabled={status === "uploading"} />
      {status === "uploading" && <p>Laddar upp...</p>}
      {status === "success" && <p>✅ Uppladdad! Nyckel: {uploadedKey}</p>}
      {status === "error" && <p>❌ Något gick fel. Försök igen.</p>}
    </div>
  );
}
```

## IAM-roller och Säkerhet

### EC2 Instance Role

Din EC2-instans bör ha en **IAM Role** med följande policy för att komma åt S3:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::min-applikation-uploads",
        "arn:aws:s3:::min-applikation-uploads/*"
      ]
    }
  ]
}
```

### Lambda Execution Role

Lambda-funktionen behöver liknande behörigheter. AWS SDK:n hämtar automatiskt credentials från Lambda-miljön.

### Bucket Policy: Blockera all direkt offentlig åtkomst

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyPublicAccess",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::min-applikation-uploads/*",
      "Condition": {
        "StringNotEquals": {
          "aws:PrincipalArn": [
            "arn:aws:iam::DITT-KONTO-ID:role/ec2-spring-boot-role",
            "arn:aws:iam::DITT-KONTO-ID:role/lambda-execution-role"
          ]
        }
      }
    }
  ]
}
```

> ⚠️ **Viktig säkerhetsregel:** Aktivera alltid **Block Public Access** på bucket-nivå om du inte har ett specifikt skäl att göra filer offentliga. Använd pre-signed URLs för att ge tillfällig åtkomst.

## CORS-konfiguration för S3

För att frontend ska kunna ladda upp direkt till S3 behöver bucketen en **CORS-konfiguration**:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": ["https://din-frontend-domän.com"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3000
  }
]
```

> Under utveckling kan du temporärt sätta `"AllowedOrigins": ["*"]`, men byt ut detta mot din riktiga domän i produktion.

### Övning 5 – Säkerhet och kostnadsoptimering (30 min)

**Mål:** Reflektera kring säkerhet och drift.

Diskutera och implementera (om tid finns) följande:

1. **Filnamnssanering:** Varför är `UUID + originalFilename` bättre än bara `originalFilename` som S3-nyckel?
