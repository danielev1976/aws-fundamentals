## What is Amazon S3?

**Amazon Simple Storage Service (S3)** is an object storage service from AWS that lets you store and retrieve any amount of data at any time, from anywhere on the web.

Unlike a file system on a computer, S3 organizes data as **objects** inside **buckets**. Each object consists of:

- **Data** – the actual file content (image, PDF, JSON, video, etc.)
- **Key** – a unique name within the bucket (acts like a file path, e.g. `uploads/users/avatar-123.png`)
- **Metadata** – optional information about the object (content-type, creation date, etc.)

### Why S3?

| Property | Description |
|---|---|
| Scalability | Store unlimited amounts of data |
| Durability | 99.999999999% (11 nines) durability |
| Cost | Pay only for what you use |
| Integration | Seamless integration with other AWS services |
| Security | Fine-grained access control via IAM and bucket policies |

---

## Core Concepts

### Bucket

A bucket is a container for objects. Every bucket has a **globally unique name** across all of AWS. You also choose which **region** (geographic location) to create the bucket in — pick one close to your users for lowest latency.

```
s3://my-application-uploads/
```

### Object Key

The key is the object's "path" within the bucket. S3 is actually flat (no real folder structure), but using `/` in key names gives a folder-like experience in the AWS Console.

```
uploads/images/profile-pictures/user-42.jpg
uploads/documents/2024/report.pdf
```

### Access Control

S3 has several layers of access control:

- **Block Public Access** – default setting that blocks all public access
- **Bucket Policy** – JSON-based rules defining who can do what
- **IAM Roles/Policies** – controls which AWS resources (EC2, Lambda, etc.) can use the bucket
- **Pre-signed URLs** – temporary, signed URLs for time-limited access

## S3 and Lambda: Pre-signed URLs

A common pattern is to let the **frontend upload directly to S3** without the file passing through your backend. This is called a **pre-signed URL upload**.

### Flow

```
1. Frontend ───POST /upload-url { filename, contentType } ──▶ Lambda
2. Lambda  ───generates pre-signed PUT URL ─────────────▶ S3
3. Lambda  ◀──returns URL (valid 15 min) ────────────────
4. Frontend ◀──{ uploadUrl: "https://s3.aws..." } ───────
5. Frontend ───PUT file.jpg directly to S3 ─────────────▶ S3
```

### Lambda Function: Generate Pre-signed Upload URL (Node.js)

```javascript
// generate-upload-url/index.mjs
import { S3Client, PutObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";
import { randomUUID } from "crypto";

const s3 = new S3Client({ region: process.env.AWS_REGION });
const BUCKET_NAME = process.env.S3_BUCKET_NAME;

export const handler = async (event) => {
  const { filename, contentType } = JSON.parse(event.body);

  // Safe key – never use the user's filename directly!
  const key = `uploads/${randomUUID()}-${filename}`;

  const command = new PutObjectCommand({
    Bucket: BUCKET_NAME,
    Key: key,
    ContentType: contentType,
  });

  // URL valid for 15 minutes
  const uploadUrl = await getSignedUrl(s3, command, { expiresIn: 900 });

  return {
    statusCode: 200,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ uploadUrl, key }),
  };
};
```

### Lambda Function: Generate Pre-signed Download URL (Node.js)

```javascript
// generate-download-url/index.mjs
import { S3Client, GetObjectCommand } from "@aws-sdk/client-s3";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";

const s3 = new S3Client({ region: process.env.AWS_REGION });

export const handler = async (event) => {
  const { key } = event.queryStringParameters;

  const command = new GetObjectCommand({
    Bucket: process.env.S3_BUCKET_NAME,
    Key: key,
  });

  const downloadUrl = await getSignedUrl(s3, command, { expiresIn: 3600 });

  return {
    statusCode: 200,
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ downloadUrl }),
  };
};
```

### S3 Configuration (Spring Boot)

```java
// config/S3Config.java
@Configuration
public class S3Config {

    @Value("${aws.region}")
    private String region;

    @Bean
    public S3Client s3Client() {
        return S3Client.builder()
                .region(Region.of(region))
                // On EC2 with an IAM Role, no explicit credentials are needed!
                // The SDK automatically fetches credentials from the EC2 Instance Metadata
                .credentialsProvider(DefaultCredentialsProvider.create())
                .build();
    }
}
```

### S3 Service (Spring Boot)

```java
// service/S3Service.java
@Service
public class S3Service {

    private final S3Client s3Client;

    @Value("${aws.s3.bucket-name}")
    private String bucketName;

    public S3Service(S3Client s3Client) {
        this.s3Client = s3Client;
    }

    /**
     * Upload a file to S3
     */
    public String uploadFile(String key, byte[] data, String contentType) {
        PutObjectRequest request = PutObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .contentType(contentType)
                .build();

        s3Client.putObject(request, RequestBody.fromBytes(data));
        return key;
    }

    /**
     * Download a file from S3
     */
    public byte[] downloadFile(String key) {
        GetObjectRequest request = GetObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build();

        ResponseBytes<GetObjectResponse> objectBytes = s3Client.getObjectAsBytes(request);
        return objectBytes.asByteArray();
    }

    /**
     * Delete an object from S3
     */
    public void deleteFile(String key) {
        DeleteObjectRequest request = DeleteObjectRequest.builder()
                .bucket(bucketName)
                .key(key)
                .build();

        s3Client.deleteObject(request);
    }

    /**
     * List all objects with a given prefix (e.g. all files for a specific user)
     */
    public List<String> listFiles(String prefix) {
        ListObjectsV2Request request = ListObjectsV2Request.builder()
                .bucket(bucketName)
                .prefix(prefix)
                .build();

        ListObjectsV2Response response = s3Client.listObjectsV2(request);
        return response.contents().stream()
                .map(S3Object::key)
                .collect(Collectors.toList());
    }
}
```

### REST Controller (Spring Boot)

```java
// controller/FileController.java
@RestController
@RequestMapping("/api/files")
public class FileController {

    private final S3Service s3Service;

    public FileController(S3Service s3Service) {
        this.s3Service = s3Service;
    }

    @PostMapping("/upload")
    public ResponseEntity<Map<String, String>> uploadFile(
            @RequestParam("file") MultipartFile file) throws IOException {

        String key = "processed/" + UUID.randomUUID() + "-" + file.getOriginalFilename();
        s3Service.uploadFile(key, file.getBytes(), file.getContentType());

        return ResponseEntity.ok(Map.of("key", key, "status", "uploaded"));
    }

    @GetMapping("/download/{key}")
    public ResponseEntity<byte[]> downloadFile(@PathVariable String key) {
        byte[] data = s3Service.downloadFile(key);

        return ResponseEntity.ok()
                .header(HttpHeaders.CONTENT_DISPOSITION, "attachment; filename=\"file\"")
                .body(data);
    }

    @DeleteMapping("/{key}")
    public ResponseEntity<Void> deleteFile(@PathVariable String key) {
        s3Service.deleteFile(key);
        return ResponseEntity.noContent().build();
    }
}
```

## Frontend: Uploading with a Pre-signed URL

```javascript
// services/fileService.js (Vue/React – works in both)

const API_BASE = "https://your-api-gateway-url.amazonaws.com";

/**
 * Step 1: Fetch a pre-signed URL from Lambda
 * Step 2: Upload the file directly to S3
 */
export async function uploadFile(file) {
  // 1. Ask Lambda for an upload URL
  const urlResponse = await fetch(`${API_BASE}/upload-url`, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
      filename: file.name,
      contentType: file.type,
    }),
  });

  const { uploadUrl, key } = await urlResponse.json();

  // 2. Upload directly to S3 using the pre-signed URL
  const uploadResponse = await fetch(uploadUrl, {
    method: "PUT",
    headers: { "Content-Type": file.type },
    body: file, // Send the file directly – no base64 encoding needed
  });

  if (!uploadResponse.ok) {
    throw new Error("Upload failed");
  }

  return key; // Save the key in your database for future retrieval
}

/**
 * Fetch a download URL to display or download a file
 */
export async function getDownloadUrl(key) {
  const response = await fetch(
    `${API_BASE}/download-url?key=${encodeURIComponent(key)}`
  );
  const { downloadUrl } = await response.json();
  return downloadUrl;
}
```

### React Component: File Upload

```jsx
// components/FileUpload.jsx
import { useState } from "react";
import { uploadFile } from "../services/fileService";

export default function FileUpload() {
  const [status, setStatus] = useState("idle");
  const [uploadedKey, setUploadedKey] = useState(null);

  const handleFileChange = async (e) => {
    const file = e.target.files[0];
    if (!file) return;

    setStatus("uploading");
    try {
      const key = await uploadFile(file);
      setUploadedKey(key);
      setStatus("success");
    } catch (error) {
      console.error(error);
      setStatus("error");
    }
  };

  return (
    <div>
      <input type="file" onChange={handleFileChange} disabled={status === "uploading"} />
      {status === "uploading" && <p>Uploading...</p>}
      {status === "success" && <p>✅ Uploaded! Key: {uploadedKey}</p>}
      {status === "error" && <p>❌ Something went wrong. Please try again.</p>}
    </div>
  );
}
```

## IAM Roles and Security

### EC2 Instance Role

Your EC2 instance should have an **IAM Role** with the following policy to access S3:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:DeleteObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::my-application-uploads",
        "arn:aws:s3:::my-application-uploads/*"
      ]
    }
  ]
}
```

### Lambda Execution Role

The Lambda function needs similar permissions. The AWS SDK automatically fetches credentials from the Lambda environment.

### Bucket Policy: Block All Direct Public Access

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "DenyPublicAccess",
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-application-uploads/*",
      "Condition": {
        "StringNotEquals": {
          "aws:PrincipalArn": [
            "arn:aws:iam::YOUR-ACCOUNT-ID:role/ec2-spring-boot-role",
            "arn:aws:iam::YOUR-ACCOUNT-ID:role/lambda-execution-role"
          ]
        }
      }
    }
  ]
}
```

> ⚠️ **Important security rule:** Always enable **Block Public Access** at the bucket level unless you have a specific reason to make files public. Use pre-signed URLs to grant temporary access instead.

## CORS Configuration for S3

For the frontend to be able to upload directly to S3, the bucket needs a **CORS configuration**:

```json
[
  {
    "AllowedHeaders": ["*"],
    "AllowedMethods": ["GET", "PUT", "POST"],
    "AllowedOrigins": ["https://your-frontend-domain.com"],
    "ExposeHeaders": ["ETag"],
    "MaxAgeSeconds": 3000
  }
]
```

> During development you can temporarily set `"AllowedOrigins": ["*"]`, but replace this with your real domain before going to production.
